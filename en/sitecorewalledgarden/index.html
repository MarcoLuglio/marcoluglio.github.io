Getting ready

Before you even think on doing anything, you will need a Sitecore license. It is what it is. If you do not work for a company that already has a deal with Sitecore, you may request a trial which will likely be followed up close by the Sitecore sales team.

Sitecore uses the .Net Framework, which is only available for Windows, and requires IIS for hosting. Sitecore 10 does have some parts written in .Net core, and an official Docker image, but because it is still bound to IIS, those are fat ass Windows images that take a lot of disk space, and require 16GB of ram.

The Docker + Sitecore 10 path is easier to get started if you have the hardware. As simple as installing Docker and running init.ps1 a command in the prompt. https://github.com/Sitecore/docker-examples/tree/develop/getting-started and https://github.com/Sitecore/Helix.Examples/tree/master/examples/helix-basic-aspnetcore are two places where you can clone starting reference projects.
